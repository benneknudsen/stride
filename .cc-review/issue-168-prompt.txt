Du skal fikse issue #168 i stride-projektet.

### Mål
- Sikre at magic-link URL aldrig kan bruges til host-poisoning, uanset om `AUTH_URL` er sat eller ej.

### Acceptkriterier
1. Tilføj en validering i `auth.config.ts` eller `lib/email.ts` der kaster/feiler hvis `NODE_ENV=production` og `AUTH_URL` ikke er sat, eller hvis Host-header ikke matcher en allowlist.
2. Ændr ikke funktionaliteten for udviklingsmiljøer (dev) — valideringen skal kun slå til i production.
3. Skriv en test der verificerer at request med spoofet host (fx `evil.example.com`) afvises i production.
4. `git add -A && git commit -m "fix(auth): beskyt magic-link mod host-poisoning (#168)"`

### Sted i koden
- Filer: `auth.config.ts:113` (trustHost), `lib/email.ts:24-26` (linkkonstruktion).
- Test: `__tests__/auth/` eller tilsvarende (kan oprettes på en passende måde).

### Constraints (fra AGENTS.md)
- TypeScript strict: ingen `any`.
- Biome 2.5: `npm run check`.
- Type-check: `npx tsc --noEmit`.
- Test: `npm test -- --run --reporter=dot`.

### Vigtigt
- Koden må ikke bryde eksisterende tests.
- Gør kun ændringer der er nødvendige for at løse issue.
